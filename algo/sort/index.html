十大排序算法

关于稳定性，这里说一下稳定性的作用：

稳定性是什么？如果 a 和 b 在数组中排序的值一样，在排序后 a 与 b 的相对位置并没有发生改变。

  • 对于基本数据类型的排序，稳定性没一点用。
  • 引用数据类型的两个字段及其更多字段进行排序时，一定需要稳定性的维持：比如一件商品需要先按好评从高到低进行排序，好评一样的话则按价格从低到高排序。
    • 我们先对价格按照升序排，比如 A 商品是 20 元 | 9.9 评分，B 商品是 30 元 | 9.9 评分，排好序后 A 商品在 B 商品的前面；
    • 接下来我们按照好评从高到低进行排序，如果不能维持稳定性，由于 A 商品和 B 商品的评分是一样的，因此 B 商品可能会排在 A 商品前面；
    • 这时候我们发现，原本排序规则是好评一样则按照价格进行升序排序，但 A 商品与 B 商品好评一样，价格却是降序排的，造成这个的原因就是因为破坏了稳定性！（JDK8 底层对引用数据类型进行排序使用的是 归并排序 + 插入排序 ，现在我们应该理解了，是因为这两个排序能够维持稳定性）

下表中的时间复杂度仅代表本文给出代码的时间复杂度，具体跟网上可能会有不一样。

希尔排序的时间复杂度跟选取的 增量序列 有关，本文给出的是较容易实现的 Knuth 增量序列，其最坏时间复杂度是 $O(n^{\frac{3}{2}})$，目前已被证实的最佳增量序列是 Pratt 序列，其最坏时间复杂度是 $O(n\log^2n)$

虽然快速排序的最坏时间复杂度是 $O(n^2)$ ，但是这种最坏情况几乎是不可能发生的，我们可以随机化选择切分元素（pivot）来降低最坏情况发生的概率，此时最坏时间复杂度发生的概率为 $\frac {2^n} {n!}$ 。此外，在三类 $O(n\log n)$ 的比较排序算法中，快排时间复杂度的常数项是最小的，所以 在能够使用快排的情况下，我们应该优先考虑使用快排。

计数数组中复杂度的表达式中 n 表示数组长度，k 表示计数数组的长度 max - min + 1，为了维持其稳定性，除了计数数组外，还需要额外开辟一个与原数组等长的数组。

七类比较排序算法

选择排序（了解）

每一轮选取未排定的部分中最小的那个元素交换到未排定部分的最开头，经过若干个步骤，就能排定整个数组。即：先选出最小的，再选出第二小的，以此类推。

Java 代码：

public class Solution {
    public void selectionSort(int[] nums) {
        if (nums == null || nums.length < 2) {
            return;
        }
        // [0, i) 有序，且该区间里所有元素就是最终排定的样子
        for (int i = 0; i < nums.length - 1; i++) {
            // 选择区间 [i, n) 里最小的元素的索引，交换到位置 i
            int minIndex = i;
            for (int j = i + 1; j < nums.length; j++) {
                if (nums[j] < nums[minIndex]) {
                    minIndex = j;
                }
            }
            swap(nums, i, minIndex);
        }
    }

    private void swap(int[] nums, int i, int j) {
        int temp = nums[i];
        nums[i] = nums[j];
        nums[j] = temp;
    }
}

复杂度及稳定性

  • 最好时间复杂度与最坏时间复杂度：$O(n^2)$
  • 空间复杂度：$O(1)$
  • 稳定性：不能保证稳定性

冒泡排序（了解）

外层循环每一次经过两两比较，把 每一轮未排定部分最大的元素放到了数组的末尾

  • 依次将相邻的两个元素进行比较，把较大的元素交换到后面，这样一轮循环下来，就可以找到这一轮循环中最大的那个元素，我们把这个过程形象地称之为“冒泡”；
  • 由于每一轮循环都「冒泡」出一个这一轮循环最大的元素，所以上一轮循环的最后一个元素，没有必要再参加下一轮循环的比较了；
  • 「冒泡排序」有个特点：在遍历的过程中，提前 检测到数组是有序的，从而结束排序，而不像「选择排序」那样，即使输入数据是有序的，「选择排序」依然需要很「死板地」地走完所有的流程。

Java 代码：

public class Solution {
    public void bubbleSort(int[] nums) {
        if (nums == null || nums.length < 2) {
            return;
        }
        // (i, n-1] 有序，且该区间里所有元素就是最终排定的样子
        for (int i = nums.length - 1; i > 0; i--) {
            // 先默认数组是有序的，只要发生一次交换，就必须进行下一轮比较
            boolean sorted = true;
            for (int j = 0; j < i; j++) {
                // 严格大于是实现稳定性的关键
                if (nums[j] > nums[j + 1]) {
                    swap(nums, j, j + 1);
                    sorted = false;
                }
            }
            // 如果在内层循环中，一次交换操作都没有执行，说明此时数组已经是升序数组
            if (sorted) {
                return;
            }
        }
    }

    private void swap(int[] nums, int i, int j) {
        int temp = nums[i];
        nums[i] = nums[j];
        nums[j] = temp;
    }
}

复杂度及稳定性

  • 最好时间复杂度：$O(n)$，最坏时间复杂度：$O(n^2)$
  • 空间复杂度：$O(1)$
  • 稳定性：可以保证稳定性

插入排序（熟悉）

每次将一个数字插入一个有序的数组里，成为一个长度更长的有序数组，有限次操作以后，数组整体有序。

Java 代码：

public class Solution {
    public void insertionSort(int[] nums) {
        if (nums == null || nums.length < 2) {
            return;
        }
        // 将 nums[i] 插入到区间 [0, i) 使之成为有序子数组
        for (int i = 1; i < nums.length; i++) {
            // 暂存要插入的元素
            int temp = nums[i];
            int j = i;
            // 严格大于 temp 是实现稳定性的关键
            while (j > 0 && nums[j - 1] > temp) {
                nums[j] = nums[j - 1];
                j--;
            }
            nums[j] = temp;
        }
    }
}

复杂度及稳定性

  • 最好时间复杂度：$O(n)$，最坏时间复杂度：$O(n^2)$
  • 空间复杂度：$O(1)$
  • 稳定性：可以保证稳定性

总结

  • 优化方向：希尔排序。
    网上所谓的二分（折半）插入排序的算法，并没有使时间复杂度降低，因为使用二分寻找到了插入位置，同样需要将该位置后（含该位置）的所有数全部右移，当然，右移的循环条件可以少一条判断。

  • 特点：「插入排序」可以提前终止内层循环（体现在 nums[j-1] > temp 不满足时），在数组「几乎有序」的前提下，「插入排序」的时间复杂度可以达到 $O(n)$；

  • 由于「插入排序」在「几乎有序」的数组上表现良好，特别地，在「短数组」上的表现也很好。因为「短数组」的特点是：每个元素离它最终排定的位置都不会太远。为此，在小区间内执行排序任务的时候，可以转向使用「插入排序」。

  • 与冒泡排序相比，虽然时间复杂度的阶数一样并且都是稳定排序，但由于插入排序在将 nums[i] 插入到区间 [0, i) 使之成为有序子数组这个过程中并不是直接使用交换，而是采用的 指针左移 方式确定 nums[i] 的位置，因此插入排序的时间复杂度常数项会优于冒泡排序。

  • 比较排序使用原则：对于短数组，优先使用插入排序，因为这时候我们更在意的是时间复杂度中的系数；而对于长数组，则优先使用快排等时间复杂度为 $O(n\log n)$ 的排序算法，因为这时候数据量大起来了，我们更在意的是时间复杂度中的阶数。但实际上快排等有着较低时间复杂度是因为使用了分治思想，因此对于长数组，我们可以设置一个阈值，先使用快排等时间复杂度优秀的排序算法，当低于这一个阈值就使用插入排序。

希尔排序（不建议多花时间了解）

插入排序的优化。在插入排序里，如果靠后的数字较小，它来到前面就得移动多次。「希尔排序」改进了这种做法。带间隔地使用插入排序，直到最后「间隔」为 1 的时候，就是标准的「插入排序」，此时数组里的元素已经「几乎有序」了；

希尔排序中的间隔指的是：比如间隔为 3，则将元素组分为如下三组

  • nums[0], nums[3], nums[6], …
  • nums[1], nums[4], nums[7], …
  • nums[2], nums[5], nums[8], …

分组的好处是在当数组长度较大时，能够有效的减少插入排序过程中的移动次数。

事实上，一般在不需要实现稳定性的情况下，我们采用 快排+插入，在快排拆分小数组时，当其数组长度低于某一个阈值，就由快排转而调用插入排序，充分利用两类排序算法的优点；而在需要实现稳定性的排序中（比如对引用类型中的字段进行排序），我们一般需要保证其稳定性，从而采用 归并+插入；在对空间复杂度要求较高的时候，我们采用 堆排序+插入排序。因此，希尔排序实际上用途并不大。

Java 代码

public class Solution {
    public static void shellSort(int[] nums) {
        if (nums == null || nums.length < 2) {
            return;
        }

        // 找增量的最大值。 使用 Knuth 增量序列: h=1; h=3*h+1 ...
        int gap = 1;
        while (3 * gap + 1 < nums.length) {
            gap = 3 * gap + 1;
        }

        // 分组间距减到 1 时，就是标准的插入排序了，再进行一次排序即可
        while (gap >= 1) {
            // 分组间隔为 gap 的插入排序
            for (int i = gap; i < nums.length; i++) {
                int temp = nums[i];
                int j = i;
                // 希尔排序的分组间隔会破坏稳定性，因此是不是严格大于都不重要了
                // （建议严格大于 -> 可能会早点结束循环）
                while (j >= gap && nums[j - gap] > temp) {
                    nums[j] = nums[j - gap];
                    j -= gap;
                }
                nums[j] = temp;
            }
            gap = gap / 3;
        }
    }
}

复杂度及稳定性

  • 希尔排序的时间复杂度跟选取的增量序列有关，本文给出的是 Knuth 增量序列，其最坏时间复杂度是 $O(N^{\frac{3}{2}})$；
  • 空间复杂度：$O(1)$
  • 稳定性：不能保证稳定性

归并排序（重点）

借助额外空间，合并两个有序数组，得到更长的有序数组。

分而治之（分治思想）。「分而治之」思想的形象理解是「曹冲称象」、MapReduce，在一定情况下可以并行化。

Java 代码：自顶向下的归并排序（推荐）

有参考 Java 中的归并排序 Arrays.mergeSort

public class Solution {
    public void mergeSort(int[] nums) {
        if (nums == null || nums.length < 2) {
            return;
        }
        mergeSort(nums, 0, nums.length - 1, new int[nums.length]);
    }

    private void mergeSort(int[] nums, int left, int right, int[] temp) {
        // 只有一个数，已然有序
        if (left == right) {
            return;
        }
        // 使用一分为二的思路，一直递归下去
        int mid = (left + right) >>> 1;
        mergeSort(nums, left, mid, temp);
        mergeSort(nums, mid + 1, right, temp);
        // 如果数组的这个子区间本身有序，无需合并
        if (nums[mid] <= nums[mid + 1]) {
            return;
        }
        mergeTwoSortedArray(nums, left, mid, right, temp);
    }

    private void mergeTwoSortedArray(int[] nums,
                                     int left,
                                     int mid,
                                     int right,
                                     int[] temp) {
        // 需要注意的是，这里为 temp 数组赋值时，索引都要从 left 开始
        System.arraycopy(nums, left, temp, left, right - left + 1);
        // p：temp 中左边数组的索引，q：temp 中右边数组的索引，i：待排序数组的索引
        for (int i = left, p = left, q = mid + 1; i <= right; i++) {
            // temp[p] <= temp[q]，这是实现稳定性的关键
            if ((q == right + 1) || (p < mid + 1 && temp[p] <= temp[q])) {
                nums[i] = temp[p];
                p++;
            } else {
                nums[i] = temp[q];
                q++;
            }
        }
    }
}

Java 代码：自底向上的归并排序

自底向上的归并排序是本人在刷 力扣148题 才知道的（本质上就是将自顶向下归并中的递归过程手动实现）

public class Solution {
    public void mergeSort(int[] nums) {
        if (nums == null || nums.length < 2) {
            return;
        }
        int[] temp = nums.clone();
        // 一一合并，两两合并，四四合并
        for (int gap = 1; gap < nums.length; gap <<= 1) {
            for (int left = 0; left + gap < nums.length; left += 2 * gap) {
                mergeTwoSortedArray(nums, left, left + gap - 1, left + 2 * gap - 1, temp);
            }
        }
    }

    private void mergeTwoSortedArray(int[] nums,
                                     int left,
                                     int mid,
                                     int right,
                                     int[] temp) {
        // 需要注意的是，这里为 temp 数组赋值时，索引都要从 left 开始
        System.arraycopy(nums, left, temp, left, right - left + 1);
        // p：temp 中左边数组的索引，q：temp 中右边数组的索引，i：待排序数组的索引
        for (int i = left, p = left, q = mid + 1; i <= right; i++) {
            // temp[p] <= temp[q]，这是实现稳定性的关键
            if ((q == right + 1) || (p < mid + 1 && temp[p] <= temp[q])) {
                nums[i] = temp[p];
                p++;
            } else {
                nums[i] = temp[q];
                q++;
            }
        }
    }
}

复杂度及稳定性

  • 时间复杂度：$O(n\log n)$
  • 空间复杂度：$O(n)$
  • 稳定性：可以保证稳定性

常见的坑（源自左程云）：

  • 归并排序的额外空间复杂度可以变成 $O(1)$，但是会破坏其稳定性，并且非常难不需要掌握，有兴趣可以搜 “归并排序” 内部缓存法；而且，既然破坏了稳定性还增大了难度，为啥不用堆排序？

  • “原地归并排序” 的帖子都是垃圾，会让归并排序的时间复杂度变成 $O(n^2)$

  • 快速排序可以做到稳定性问题，但是其空间复杂度会变成 $O(n)$，并且非常难不需要掌握，可以搜 “01 stable sort”。而且，既然空间复杂度增到了 $O(n)$ 还增大了难度，为啥不用归并?

快速排序（重点）

快速排序每一次都排定一个元素（这个元素待在了它最终应该待的位置），然后递归地去排它左边的部分和右边的部分，依次进行下去，直到数组有序；

分而治之（分治思想），与「归并排序」不同，「快速排序」在「分」这件事情上不像「归并排序」无脑地一分为二，而是采用了 partition 的方法，因此就没有「合」的过程。

实现细节：一定要随机化选择切分元素（pivot），降低最坏情况下 $O(n^2)$ 时间复杂度的发生。

Java 代码：单轴快排

把等于切分元素的所有元素分到了数组的同一侧，可能会造成递归树倾斜

public class Solution {
    public void quickSort(int[] nums) {
        if (nums == null || nums.length < 2) {
            return;
        }
        quickSort(nums, 0, nums.length - 1);
    }

    private void quickSort(int[] nums, int left, int right) {
        // 随机选取的基准值最终索引可能等于 left 或 right，故不能简单的直接取等号
        // 一个行之有效的办法是：一律取成大于等于。能取等号的我代码中都写的等于
        if (left >= right) {
            return;
        }
        int picked = partition(nums, left, right);
        quickSort(nums, left, picked - 1);
        quickSort(nums, picked + 1, right);
    }

    /**
     * 在数组 nums 的子区间 [left, right] 中随机选取一个基准值将其放入它最终应该待的位置
     *
     * @return 随机选取的基准值最终所在位置索引
     */
    private int partition(int[] nums, int left, int right) {
        // 生成一个位于 [left, right] 中的随机整数
        int picked = (int) (Math.random() * (right - left + 1) + left);
        swap(nums, picked, right);
        // 基准值
        int pivot = nums[right];

        // nums[left, lt) < pivot
        // nums[lt, i) >= pivot
        int lt = left;
        for (int i = left; i < right; i++) {
            if (nums[i] < pivot) {
                swap(nums, lt, i);
                lt++;
            }
        }
        swap(nums, lt, right);
        return lt;
    }

    private void swap(int[] nums, int i, int j) {
        int temp = nums[i];
        nums[i] = nums[j];
        nums[j] = temp;
    }
}

Java 代码：双轴快排

把等于切分元素的所有元素挤到了数组的中间，在有很多元素和切分元素相等的情况下，递归区间大大减少。

public class Solution {
    public void quickSort(int[] nums) {
        if (nums == null || nums.length < 2) {
            return;
        }
        quickSort(nums, 0, nums.length - 1);
    }

    private void quickSort(int[] nums, int left, int right) {
        if (left >= right) {
            return;
        }
        int[] partition = partition(nums, left, right);
        // 注意这里，大大减少了两侧分治的区间
        quickSort(nums, left, partition[0]);
        quickSort(nums, partition[1], right);
    }

    /**
     * 在数组 nums 的子区间 [left, right] 中随机选取一个基准值将其放入它最终应该待的位置
     *
     * @return 第一个元素是小于基准值的最大索引，第二个元素是大于基准值的最小索引
     */
    private int[] partition(int[] nums, int left, int right) {
        // 生成一个位于 [left, right] 中的随机整数
        int picked = (int) (Math.random() * (right - left + 1) + left);
        swap(nums, picked, right);
        // 基准值
        int pivot = nums[right];

        // nums[left, lt) < pivot
        // nums[lt, i) = pivot
        // nums(gt, right-1] > pivot
        int lt = left;
        int gt = right - 1;
        label:
        for (int i = left; i <= gt; i++) {
            // 如果 nums[i] > pivot ，则交换。直到 i 位置的数 <= pivot
            while (nums[i] > pivot) {
                swap(nums, i, gt);
                gt--;
                if (i > gt) {
                    break label;
                }
            }
            // nums[i] < pivot
            if (nums[i] < pivot) {
                swap(nums, i, lt);
                lt++;
            }
        }
        // 处理基准值（right位置上的数）
        swap(nums, right, gt + 1);
        return new int[]{lt - 1, gt + 2};
    }

    private void swap(int[] nums, int i, int j) {
        int temp = nums[i];
        nums[i] = nums[j];
        nums[j] = temp;
    }
}

堆排序（重点）

堆排序是选择排序的优化，选择排序需要在未排定的部分里通过「打擂台」的方式选出最大的元素（复杂度 $O(n)$），而「堆排序」就把未排定的部分构建成一个「大顶堆」，这样就能以 $O(\log⁡ n)$ 的方式选出最大元素；

堆是一种相当有意思的数据结构，它在很多语言里也被命名为「优先队列」Java 中的 PriorityQueue 就是堆排序。它是建立在数组上的「树」结构，类似的数据结构还有「并查集」「线段树」等。

堆化的时间复杂度分析

堆中元素个数 2k-1 - 1 < n <= 2k - 1 ，即二叉树的高度为 k ，注意到第 i 层结点数为 2i-1 ，这在计算时间复杂度时要用上。

向上堆化

一个深度为 i （第 i 层）的结点堆化需要 i 次判断

$T(2^k-1) = \sum\limits_{i=1}^{k} i\times 2^{i-1} = 1 \times 2^0 + 2 \times 2^1 + 3 \times 2^2 + \cdots + k \times 2^{k-1} = (k-1)2^k + 1$

于是：

$T(n) \le T(2^k-1) = (k-1)2^k+1 < \log(n+1) \times 2(n+1)+1 = O(n\log n)$

$T(n) > T(2^{k-1}-1) = (k-2)2^{k-1}+1 \ge (\log(n+1)-2) \times 2^{\log (n+1)-1} + 1 = O(n\log n)$

从而向上堆化的时间复杂度是 $O(n\log n)$

for (int i = 0; i < nums.length; i++) {
    siftUp(nums, i);
}

private void siftUp(int[] nums, int k) {
    // parent 父节点的下标
    for (int parent = (k - 1) >>> 1; nums[k] > nums[parent]; parent = (k - 1) >>> 1) {
        swap(nums, k, parent);
        k = parent;
    }
}

向下堆化

一个深度为 i （第 i 层）的结点堆化需要 k + 1 - i 次判断

$T(2^k-1) = \sum\limits_{i=1}^{k} (k+1-i)\times 2^{i-1} = k \times 2^0 + (k-1) \times 2^1 + \cdots + 1 \times 2^{k-1} = 2^{k+1} -2 -k$

于是：

$T(n) \le T(2^k-1) = 2^{k+1}-2-k < 4(n+1)-2-\log(n+1) = O(n)$

$T(n) > T(2^{k-1}-1) = 2^{k}-2-(k-1) > (n+1)-2-\log(n+1) = O(n)$

从而向上堆化的时间复杂度是 $O(n)$

private void heapify(int[] nums) {
    for (int i = nums.length - 1; i >= 0; i--) {
        siftDown(nums, i, nums.length);
    }
}

private void siftDown(int[] nums, int k, int heapSize) {
    // left 左孩子下标
    for (int left = (k << 1) + 1; left < heapSize; left = (k << 1) + 1) {
        // 右孩子下标
        int right = left + 1;
        // largest 存储左右孩子中最大值的下标
        int largest = (right < heapSize && nums[right] > nums[left]) ? right : left;
        // 当前结点的值不小于两个孩子的值，可以结束了
        if (nums[k] >= nums[largest]) {
            return;
        }
        // 当前结点值小于孩子之间的最大值，需要交换
        swap(nums, k, largest);
        k = largest;
    }
}

因此，堆化过程我们可以选择更优的向下堆化，也即 heapify 。（但这不影响总的时间复杂度）

public class Solution {
    public void heapSort(int[] nums) {
        if (nums == null || nums.length < 2) {
            return nums;
        }
        // 构造大顶堆（最大堆）
        heapify(nums);
        // 每一轮将堆顶元素与堆中最后一个元素进行交换，然后维护大顶堆。
        for (int heapSize = nums.length - 1; heapSize >= 0; heapSize--) {
            swap(nums, 0, heapSize);
            siftDown(nums, 0, heapSize);
        }
    }

    /**
     * 构造大顶堆
     */
    private void heapify(int[] nums) {
        for (int i = nums.length - 1; i >= 0; i--) {
            siftDown(nums, i, nums.length);
        }
    }

    /**
     * 在位置 k 处插入项 nums[k]，并维护大顶堆。
     */
    private void siftDown(int[] nums, int k, int heapSize) {
        // left 左孩子下标
        for (int left = (k << 1) + 1; left < heapSize; left = (k << 1) + 1) {
            // 右孩子下标
            int right = left + 1;
            // largest 存储左右孩子中最大值的下标
            int largest = (right < heapSize && nums[right] > nums[left]) ? right : left;
            // 当前结点的值不小于两个孩子的值，可以结束了
            if (nums[k] >= nums[largest]) {
                return;
            }
            // 当前结点值小于孩子之间的最大值，需要交换
            swap(nums, k, largest);
            k = largest;
        }
    }

    private void swap(int[] nums, int i, int j) {
        int temp = nums[i];
        nums[i] = nums[j];
        nums[j] = temp;
    }
}

复杂度及稳定性

  • 时间复杂度：$O(n\log n)$
  • 空间复杂度：$O(1)$
  • 稳定性：不能保证稳定性

三类非比较排序算法

  • 这三种排序的区别与上面的排序的特点是：一个数该放在哪里，是由这个数本身的大小决定的，它不需要经过比较。也可以认为是哈希的思想：由数值映射地址。因此这三种算法一定需要额外的空间才能完成排序任务，时间复杂度可以提升到 $O(n)$，但适用场景不多，主要是因为 使用这三种排序一定要保证输入数组的每个元素都在一个合理的范围内。

  • 这三种算法还有一个特点是：都可以实现成稳定排序，无需稳定化。

计数排序（熟悉）

「计数排序」是这三种排序算法里最好理解的，从名字就可以看出。把每个出现的数值都做一个计数，然后根据计数从小到大输出得到有序数组。

  • 先开辟一个数组长度为 max{nums[i] + 1} 的计数数组 counts，并将每个元素的初始值都初始化为零；
  • nums[i] = j，则 counts[j]++;

这种做法丢失了稳定性，如果是本题这种基本数据类型的话没有关系。如果是对象类型，就不能这么做了。

保持稳定性的做法是：先对计数数组做前缀和，在往回赋值的时候，根据原始输入数组的数据从后向前赋值，前缀和数组保存了每个元素存放的下标信息。

处理元素中可能出现的负数

上面给出的是一般情况下计数排序的实现原理（数组中的元素值不能为负数），为了兼容负数，下面给出两种解决方案。

方案一：

  • 先求出数组的最小元素 min 与最大元素 max;
  • 对原数组进行一个预处理，将原数组中每个元素都减去 min，即可得到一个新的待排序数组；
  • 开辟一个数组长度为 max - min + 1 的计数数组 counts，并将每个元素的初始值都初始化为零，在新的待排序数组上使用计数排序;
  • 将排好序的新数组中每个元素都加上预处理过程减去的 min，即可还原为排好序的原数组。

方案二：

  • 先求出数组的最小元素 min 与最大元素 max;
  • 开辟一个数组长度为 max - min + 1 的计数数组 counts，并将每个元素的初始值都初始化为零；
  • nums[i] = j，则 counts[j - min]++;

显然，方案一代码可读性更强一点，但相对的会增加部分代码量。下面给出的代码是基于方案二的

后面的基数排序和桶排序也是使用的相同的处理办法，因此后文就不再做特别说明了。

Java 代码：

public class Solution {
    public void countingSort(int[] nums) {
        if (nums == null || nums.length < 2) {
            return;
        }

        // 数组最大值、最小值
        int max = Arrays.stream(nums).max().getAsInt();
        int min = Arrays.stream(nums).min().getAsInt();
        // 计数数组
        int[] counts = new int[max - min + 1];
        // 计算计数数组
        for (int num : nums) {
            counts[num - min]++;
        }

        // 把 count 数组变成前缀和数组
        for (int i = 1; i < counts.length; i++) {
            counts[i] += counts[i - 1];
        }

        // 先把原始数组赋值到一个临时数组里，然后回写数据
        int[] temp = nums.clone();

        // 为了保证稳定性，从后向前赋值
        for (int i = nums.length - 1; i >= 0; i--) {
            // counts[temp[i] - min] ----> 原来对应的是值为 temp[i] 的个数
            // 改为前缀和数组后，其表示的是 nums 数组排好序后最后一个值为 temp[i] 的存放的位置
            // 如果数组中存在多个值等于 temp[i]，由于我们从后向前赋值，并不会破坏稳定性
            nums[--counts[temp[i] - min]] = temp[i];
        }
    }
}

注意：网上不少计数排序使用的是空间复杂度为 $O(k)$ 的算法，该算法不能够实现其稳定性。

  • 时间复杂度：$O(n+k)$
  • 空间复杂度：$O(n+k)$
  • 稳定性：可以保证稳定性

低配版计数排序

上面给出的是一个全能（能统计负数，并且能保证稳定性）的计数排序。事实上，我们在使用计数排序时，很多时候是不需要这两个功能，因此下面给出了简单版的计数排序（无稳定性、值不能为负数）。

public class Solution {
    public void countingSort(int[] nums) {
        if (nums == null || nums.length < 2) {
            return;
        }

        int max = Arrays.stream(nums).max().getAsInt();
        int[] counts = new int[max + 1];
        for (int num : nums) {
            counts[num]++;
        }

        int index = 0;
        for (int i = 0; i < counts.length; i++) {
            while (counts[i] > 0) {
                nums[index++] = i;
                counts[i]--;
            }
        }
    }
}

基数排序（了解）

也称为基于关键字的排序，例如针对数值排序，个位、十位、百位就是关键字。针对日期数据的排序：年、月、日、时、分、秒就是关键字。

「基数排序」用到了「计数排序」。

Java 代码

下面的代码中计数数组长度为 10，即 k = 10；（k = 10 是代码中给出的是基于十进制的），即：

  • 先按个位上的数进行排序，排序后的数组再按照十位上的数进行排序 … 直到最高位排完即整个数组有序。
public class Solution {
    public void radixSort(int[] nums) {
        if (nums == null || nums.length < 2) {
            return;
        }

        // 数组的最大、最小值
        int max = Arrays.stream(nums).max().getAsInt();
        int min = Arrays.stream(nums).min().getAsInt();

        // 计算出最大的数字有几位，这个数值决定了我们要将整个数组看几遍
        int maxLen = String.valueOf(max - min).length();
        for (int i = 0, divisor = 1; i < maxLen; i++, divisor *= 10) {
            // 计数数组
            int[] counts = new int[10];
            for (int num : nums) {
                counts[(num - min) / divisor % 10]++;
            }
            // 把 count 数组变成前缀和数组
            for (int j = 1; j < counts.length; j++) {
                counts[j] += counts[j - 1];
            }
            // 有多少个数准备多少个辅助空间
            int[] bucket = nums.clone();
            // 为了保证稳定性，从后向前赋值
            for (int j = nums.length - 1; j >= 0; j--) {
                nums[--counts[(bucket[j] - min) / divisor % 10]] = bucket[j];
            }
        }
    }
}

复杂度及稳定性

在本文实现中基数排序较计数排序的优势是减少了部分的空间复杂度（计数数组 k 变小了），值得一提的是，网上不少基数排序的代码实现使用的是二维数组，其空间复杂度为 $O(n \times k)$，在与计数排序比较中时间复杂度不占优的情况下（需要进行多轮排序），其空间复杂度可能(由于 k 变小了，因此此时的 n * k 不一定大于计数排序中的 n+k)也不占优，图啥呢？

  • 时间复杂度：$O((n + k)\times maxLen)$
  • 空间复杂度：$O(n+k)$
  • 稳定性：可以保证稳定性

桶排序（了解）

一个坑一个萝卜，也可以一个坑多个萝卜，对每个坑排序，再拿出来，整体就有序。

Java 代码

public class Solution {
    public void bucketSort(int[] nums) {
        if (nums == null || nums.length < 2) {
            return;
        }

        // 数组的最大、最小值
        int max = Arrays.stream(nums).max().getAsInt();
        int min = Arrays.stream(nums).min().getAsInt();
        // 设置步长，根据步长确定桶的个数
        int step = 10;
        int bucketCount = ((max - min + 1) + step - 1) / step;

        // 由 bucketCount 个一维数组组成的二维数组，每个一维数组是一个桶
        int[][] bucket = new int[bucketCount][nums.length];
        // counts[i] 表示 bucket[i] 这个桶中元素个数
        int[] counts = new int[bucketCount];

        // 分桶
        for (int num : nums) {
            int index = (num - min) / step;
            bucket[index][counts[index]++] = num;
        }

        // 对于每个桶执行插入排序
        for (int i = 0; i < bucketCount; i++) {
            insertionSort(bucket[i], counts[i] - 1);
        }

        // 从桶里依次取出来
        for (int i = 0, index = 0; i < bucketCount; i++) {
            for (int j = 0; j < counts[i]; j++) {
                nums[index++] = bucket[i][j];
            }
        }
    }

    /**
     * 对数组 nums 的子区间 [0, endIndex] 使用插入排序
     */
    private void insertionSort(int[] nums, int endIndex) {
        for (int i = 1; i <= endIndex; i++) {
            int temp = nums[i];
            int j = i;
            while (j > 0 && nums[j - 1] > temp) {
                nums[j] = nums[j - 1];
                j--;
            }
            nums[j] = temp;
        }
    }
}
